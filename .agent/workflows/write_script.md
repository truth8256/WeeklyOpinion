이 워크플로는 월 1회 유튜브 방송 "Weekly Opinion" 대본 작성 전 과정을 안내한다.
사회자와의 대화 형식으로, 전문가(김진실 수석연구원)가 여론조사 수치를 직접 설명하고 의미를 짚어주는 구조다.

---

## 1. 주제 및 자료 패키징

- **이번 방송의 메인 테마**를 확정한다. 정기 지지율 소개보다 특정 이슈 심층 분석을 우선한다.
  - 예: 검찰 개혁 여론, 특정 선거 결과 분석, 사회적 갈등 현안 등
- 테마에 직결된 **여론조사 데이터**를 수집·정리한다.
  - 조사기관, 조사일자, 표본 수, 오차범위, 중선관위 공표 여부를 확인한다.
  - 세부 집단(성별·연령·지역·지지정당·이념성향)별 교차 데이터가 있으면 함께 확보한다.
- **정기 여론조사(지지율) 포함 여부**를 판단한다.
  - 급격한 변동이 있거나 명시적으로 요청된 경우에만 포함한다.
- 수집한 자료를 **하나의 패키지**로 묶어 정리한다(수치 목록, 출처, 핵심 발견).

---

## 2. 개요 설계

- 전체 방송 흐름을 섹션 단위로 설계한다.
  - 섹션 순서: 메인 테마 → (선택) 정기 여론조사 → 마무리
  - 메인 테마가 2개 이상이면 각각 별도 섹션으로 분리한다.
  - 하나의 섹션 내에서 비교 축이 다른 분석(예: 시계열 vs 집단별)은 하위 섹션(4-1, 4-2 등)으로 나눈다.
- 각 섹션별로 다음을 간략히 정한다.
  - 사회자의 **도입 질문** 키워드
  - 전달할 **핵심 메시지** 1~2문장
  - 활용할 **데이터 포인트** (수치, 집단 비교, 시계열 변화 등)
  - 연동할 **슬라이드 ID** 또는 슬라이드 내용 (4단계에서 구체화)
- 전체 분량과 흐름이 방송 시간에 맞는지 검토한다.
- **개요 초안을 사용자에게 먼저 제시하고 피드백을 받은 뒤 3단계로 넘어간다.**
  - 섹션 순서, 포함/제외할 데이터 항목, 강조점 등 구조적 결정을 이 단계에서 확정한다.
  - 사용자가 섹션 구조를 수정하면 반영 후 재확인 없이 바로 3단계로 진행한다.

## 3. 대본 초안

- `scripts/` 폴더에 새 마크다운 파일을 생성한다.
  - 파일명: `Weekly Opinion [날짜 또는 월주].md` (예: `Weekly Opinion 260219.md`)
  - `scripts/template.md`는 포맷·문체 참고용이다. 섹션 구성은 2단계에서 확정한 개요를 따르며, 템플릿 구조를 그대로 복사하지 않는다.
- 개요를 바탕으로 섹션별 대본을 작성한다.

**작성 원칙**
- 도입부는 **2문단 이내**로 간결하게 유지한다. 전체 대본의 10%를 넘지 않는다.
  - 1문단: 배경 사건·발언 등 현안 소개 (관심 유발)
  - 2문단: 핵심 제도·맥락 설명 + 오늘 다룰 내용 예고
- 사회자 질문(`✅ **(진행자 질문)**`)과 전문가 답변을 교대로 구성한다.
- 숫자는 정확히 거론하되, 가능하다면 **비교·변화·맥락**과 함께 제시한다.
  - 나쁜 예: "긍정 60%대입니다."
  - 좋은 예: "취임 직후 62.7%였던 지지율이 반년이 지난 지금도 62.2%로 거의 그대로입니다."
- 섹션에서 처음 언급하는 조사는 **조사기관, 조사일, 표본 수**를 함께 밝힌다.
  - 예: "MBC(2월 13일 보도)와 SBS(2월 12~14일 조사) — 두 조사 모두 전국 1,000명 규모입니다."
- 핵심 수치가 3개 이상 나열되는 경우 **마크다운 표**로 정리해 대본에 삽입한다.
  - 표는 슬라이드 설계의 기초 자료로도 활용된다.
- 분석은 수치 나열이 아니라 **의미와 전망** 중심으로 전개한다.
- 집단별 차이가 있을 경우, 가장 대비가 뚜렷한 집단을 먼저 언급한다.
- 마무리는 이번 회 핵심 메시지를 1~2문장으로 정리하고, 다음 방송을 예고하는 문장으로 끝낸다.

## 4. 슬라이드 연동

- 대본의 각 섹션이 `index.html`의 어느 슬라이드와 대응되는지 확인한다.
  - 슬라이드 순서는 `slideOrder` 배열 기준 (index.html 내 `/* 슬라이드 순서 */` 주석 참조).
- 기존 슬라이드로 커버되지 않는 데이터가 있으면 신규 슬라이드 추가를 검토한다.
  - HTML에 `.slide-container` div 추가 → `slideOrder` 배열에 ID 삽입 → 차트 함수 작성.
- 데이터 변경이 필요하면 `data/` 폴더의 CSV 파일을 수정한다.
  - 새 조사 결과는 해당 CSV 마지막 행에 추가한다.
  - 조사 출처 문구(`SOURCE_WEEKLY_*` 상수)도 최신 조사일 기준으로 업데이트한다.
- 대본의 슬라이드 전환 타이밍(어느 발화에서 다음 슬라이드로 넘어가는지)을 메모해둔다.

## 5. 최종 검토

**리듬 점검**
- [ ] 너무 길거나 반복되는 부분이 있는가
- [ ] 수치가 빠진 부분 있는가
- [ ] 해석이 과도하게 단정적인 부분이 있는가

**내용 점검**
- [ ] 도입부에서 주제에 대한 관심을 적절히 유발하고 있는가
- [ ] 각 섹션의 핵심 메시지가 명확하게 전달되는가?
- [ ] 조사 출처(기관명, 조사일)가 대본 또는 슬라이드에 명시되어 있는가?
- [ ] 수치 오기 없는가?

**슬라이드 점검**
- [ ] 대본 흐름과 슬라이드 순서가 일치하는가?
- [ ] CSV 데이터가 최신 조사 결과로 업데이트되었는가?
- [ ] `index.html`을 Chrome에서 열어 차트가 정상 표시되는가?
